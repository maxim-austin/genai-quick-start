GenAI solutions can be expensive - especially when using state-of-the-art models (e.g., Claude Opus) and multi-agent workflows. Processing a single complex, research-type question can easily cost $5+ per run (per-token inference cost only). Some stakeholders will react with: "That's too much – imagine how many users and questions we'll have."

## Shift this discussion from "cost per question" to ROI

- How long would it take a user (or analyst) to produce the same result manually?
- What is the fully loaded labor rate for that role?
- What is the value of faster cycle time, better consistency, and reduced rework?

This framing helps you either justify the cost or quickly discover that the chosen use case is not economically viable (or needs a different solution approach).

## Recommendations

- **Before implementation, clearly define the business use cases** you are targeting. Identify user personas, their goals and concerns, and define success criteria and measurable metrics that enable ROI calculation (time saved, reduced manual effort, faster decision cycles, etc.).
- **Ensure you can collect and report detailed cost metrics** (per request, per workflow, per user/persona, and per use case) and communicate these costs transparently to stakeholders.
- **Plan for cost optimization as a deliberate phase.** Once you achieve target accuracy and user trust, pursue cost reduction through options such as using cheaper models where possible, managing the context window effectively, reducing unnecessary agent steps, caching/reuse, and routing "simple" questions to lower-cost paths.
